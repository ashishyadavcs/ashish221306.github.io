Angular Commands

Angular CLI makes it easy to start with any Angular project. Angular CLI comes with commands that help us create and start on our project very fast. Let us now go through the commands available to create a project, a component and services, change the port, etc.

To work with Angular CLI , you need to have installed it on our system. Let us use the following command for the same âˆ’ npm install -g @angular/cli

TO create new project you can run following command in command prompt or IDE terminal preferred -> Visual Studio Code ng new nameofapp to run your project go to your project directory and run following command - ng serve ng serve will compile your app and you can see output in your browser on https://localhost:4200 to open your project automtically in browser run following command- ng serve --open

4200 is the default port when you run a new project , you can change this default port by following command

ng serve --host 0.0.0.0 --port newport
eg. ng serve --host 0.0.0.0 --port 3000

Important commands

To generate new command
component ng g component componentname
Directive ng g directive new-directive
Pipe ng g pipe new-pipe
Module ng g module my-module
ng add It is used to add support for an external library to your project.
ng build It compiles an Angular app into an output directory named dist/ at the given output path. Must be executed from within a workspace directory.
new It creates a new workspace and an initial Angular app.

All Angular CLI commands

  1. ng add

    ng add [collectionname]
    colection name defines the lirary name which you want to add

    ng add command is used to add an external library . It adds a package for a pulished npm library. and makes your default app to use that library.

  2. ng build

    ng build

    Ng build command is used to generate project for production build. so that you may upload it on server

  3. ng config

    ng config json-path value

    jsonPath: The configuration key to set or query, in JSON path format. For example: "a[3].foo.bar[2]". If no new value is provided, returns the current value of this key.
    value: If provided, a new value for the given configuration key.

    ang config command is used to configure you angular application in angular.json file.

  4. ng e2e

    ng e2e project
    project -> It specifies the name of the project you want to build. It can be an app or a library.

    It is used to build and serve an Angular app, then runs end-to-end tests using Protractor.

  5. ng generate class

    ng generate class or ng ge class

  6. ng generate directive

    ng generate directive
    ng g directive

    It is used to create a new generic directive definition in the given or default project.

  7. ng generate service servicename

    ng generate service servicename or ng g s servicename

    It is used to create a new, generic service definition in the given or default project.
